Values and formulas

Plain text and numbers are stored directly. Punctuation alone does not turn a label into R code. Begin spreadsheet-style R formulas with =, especially when an expression could be ambiguous. A braced formula may contain several R statements; the final value is displayed.

Text cells support minimal inline Markdown: **bold** or __bold__, and *italic* or _italic_. The original cell input is preserved.

=1 + 2
=mean(ref("A1:A10"), na.rm = TRUE)
={
  x <- ref("B2#")
  sum(x)
}

Vectors spill down. Matrices, arrays, and data frames spill across rows and columns. A blocked spill reports #SPILL!.

ref()

PatternMeaning
ref("A1")One cell using A1 notation.
ref("R4C2")One cell using R1C1 notation.
ref("A1:G5")A rectangular range.
ref("B8#")The complete dynamic spill rooted at B8.
ref("'Data & Pivot'!A5#")A cross-sheet spill. Quote sheet names containing spaces.
ref("my_name")A Name manager range, expression, constant, list, or function.

References are case-insensitive. Exact spill references recover supported original R objects, not merely formatted cell text. This allows operations such as ref("A5#")$column or calling a stored function with ref("B12")(10).

Objects and element names

RGrid preserves functions, environments, lists, matrices, arrays, data frames, names, dimensions, and classes where possible. Double-click a list cell to inspect an expandable tree.

Show element names controls visible vector names, row names, column names, and dimension names. This setting is stored with the workbook.

Packages: library() and ::

RGrid recognizes package use in formulas written with library(package), library("package"), require(package), and qualified calls such as package::function(). In the browser, a referenced package is installed automatically when necessary before the formula is evaluated.

All worksheets and cells use the same R session. After a cell containing, for example, =library(data.table) has evaluated successfully, data.table is attached to the shared search path. Its exported functions can then be called without a prefix from cells in any sheet. This is ordinary R session-wide library() behaviour, not a sheet-local setting.

Calculation order still matters. On a fresh or complete recalculation, an independent formula may run before the library() cell. Merely placing the loading cell above another cell or on an earlier sheet does not create a dependency. Add an explicit ref() dependency, or use qualified calls.

package::function() calls an exported function from the package namespace without attaching the package. It therefore avoids search-path conflicts and does not depend on another cell having run first. For reproducible workbook formulas, qualified calls such as data.table::dcast() are usually the safer choice.

# Setup!A1
=library(data.table)

# Another cell with an explicit dependency
={
  ref("'Setup'!A1")
  dcast(long_data, product ~ quarter, value.var = "sales", fun.aggregate = sum)
}

# No attachment or dependency on a setup cell
=data.table::dcast(long_data, product ~ quarter, value.var = "sales", fun.aggregate = sum)

Exported RGrid scripts collect package dependencies from both styles and install missing packages before evaluating workbook cells. Once installed, the formulas follow the same library() and :: semantics as a standard R session.

Import and export

Tabular files

Import data accepts CSV, TSV, TXT, XLS, and XLSX files. Imported tables become new worksheets. Formula-looking text, including text beginning with =, remains literal text.

RGrid R scripts

Export to RGrid R script creates the lossless RGrid workbook format and an executable R script. The script installs missing packages with install.packages() when they are unavailable, so the exported file can be sourced by a standard R interpreter.

source("RGrid-Workbook.R")
str(rgrid$values)
str(rgrid$objects)

Use Import from RGrid R script to restore the exact workbook. CSV and Excel exports contain calculated values only.

Plots

Base graphics are captured from formulas. Returned ggplot2 and lattice objects are printed automatically and retained as objects. Their cells display Plot with ggplot2 or lattice in superscript and a small plot marker in the top-left corner.

={ plot(1:10, (1:10)^2, type = "b"); invisible(NULL) }
=ggplot2::ggplot(mtcars, ggplot2::aes(wt, mpg)) + ggplot2::geom_point()
=lattice::xyplot(mpg ~ wt | factor(cyl), data = mtcars)

Use Set plot size to control bitmap dimensions, and use the divider to resize the plot pane.
